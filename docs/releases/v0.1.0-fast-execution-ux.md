# EGA House Platform v0.1.0 — Fast Execution UX

Release date: 2026-05-01

## Summary

This release closes the Sprint 1 Fast Execution UX work for EGA House Platform. It focuses on making task capture, execution, Kanban flow, and saved views faster and safer without introducing parallel systems.

The release keeps the existing task creation path, task session model, Kanban workspace, and saved-view infrastructure while adding command-style capture, compact execution cards, default execution views, and final release validation.

## Highlights

### Quick Add command capture

- Added a shell/sidebar Quick Add command flow for creating tasks from one line.
- Command parsing supports task title, project token, due date, priority, and estimate.
- Added Goal token support with `/Goal` and `goal:Goal Name` syntax.
- Added blocked task token support with `@blocked:reason`.
- Goal resolution is constrained to the selected or parsed project.
- Unknown or cross-project Goals produce clear validation errors.
- Blocked tasks require a non-empty blocked reason.
- Quick Add submits through the existing task creation path rather than a parallel creation surface.

### Worked-time task creation path

- Create Task can include optional already-worked From/To fields.
- Manual worked-time input is validated and normalized.
- Valid worked-time intervals create completed task sessions linked to the new task.
- Logged sessions flow through the existing timer surfaces, totals, history, distribution, and CSV export paths.

### Compact Kanban execution cards

- Kanban cards now keep the primary execution view compact.
- Timer handoff remains visible for active, non-completed tasks.
- Secondary metadata and lifecycle controls live in a Details disclosure.
- Details expose Goal, estimate, tracked time, full blocked reason, and lifecycle actions.
- Blocked cards have distinct visual treatment and a truncated blocked-reason preview.
- Full blocked reasons are readable in Details.
- Status moves, blocked reason flow, pin/unpin, archive, restore, delete, and timer actions preserve the current Kanban return path.

### Default saved views

Added system default saved views backed by JSON definitions rather than seeded per-user rows:

- Deep Work — active urgent/high-priority tasks estimated at least 30 minutes.
- Quick Wins — active tasks estimated at 15 minutes or less.
- Blocked — active blocked tasks.
- Due This Week — active tasks due today through the next 7 days.

Saved-view links preserve List/Kanban layout, and custom saved views continue to work.

### Saved views panel polish

- Default views now appear above custom views.
- Default views are grouped under a dedicated section and labeled Built-in.
- Custom views remain grouped separately.
- Custom views keep saved counts, descriptions, open links, update controls, and delete controls.
- The save-current-view form remains available even when there are no custom views.
